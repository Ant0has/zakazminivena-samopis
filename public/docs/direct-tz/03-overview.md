### ⚠ Решение Антона 2026-05-17: исключить Сочи (AER) из приоритетов рекламы

Сочи — лидер по поисковому спросу (Wordstat показал суммарно 5 611 freq на топ-маршрутах AER → Абхазия / Гагра / Красная Поляна / Сухум / Роза Хутор), **но в Сочи дефицит машин и водителей** — масштабный рекламный трафик не сможем обслужить.

**Что делаем:**
- Страницы `/airport/aer/*` и `/destination/krasnodar-coast/*` остаются в проде для **органического трафика** — там нет давления по объёму.
- В Директе: **не запускать платные ключи по Сочи/Абхазии/Гагре** в кампании `AIRPORT_REGIONS` — добавить эти регионы в **минус-слова кампании** (или вообще убрать группы `airport_aer` из импорта `direct_keywords.csv`).
- Бюджет, который был зарезервирован под AER (~300 ₽/день), перераспределить: +200 ₽ в `AIRPORT_MOSCOW` (SVO → Тверь — 1445 freq, недооценённый), +200 ₽ в `CITIES` и `DESTINATIONS`.
- Если в будущем парк в Сочи восстановится — пересмотреть решение.

## 6. Стратегии и корректировки

**Стратегии по кампаниям:**
- AIRPORT_MOSCOW, AIRPORT_REGIONS, CITIES → **Оплата за конверсии** (`form_submit`), стоимость конверсии 800–1 200 ₽
- DESTINATIONS, SERVICES, B2B → **Оплата за клики** (low CTR, нужен трафик для обучения)
- BRAND → **Оплата за клики** (минимум, защита бренда)
- FLEET → ставка вручную

**Корректировки ставок:**
| Условие | Корректировка |
|---|---|
| Мобайл | +10% (88% трафика мобайл) |
| Аудитория «scroll_to_price без формы» | +50% |
| Аудитория «открыли форму без отправки» | +100% |
| Аудитория «бросили карзину» 28 дней | +30% |
| Возраст 25–55 + женщины (семьи) | +10% |
| Праздничные дни (31 декабря – 10 января) | +20% |

**Регионы показа:**
- AIRPORT_MOSCOW → Москва + МО + все регионы РФ
- AIRPORT_REGIONS → Россия + СНГ
- DESTINATIONS → Россия + города-доноры (Москва, СПб для Карелии и т.д.)
- CITIES → конкретный город + соседи
- B2B → Россия

**Расписание:**
- Будни 7:00–24:00, выходные 8:00–22:00
- Ночь 00:00–06:00 — отключить везде кроме AIRPORT_MOSCOW (ночные рейсы)

## 9. Доработки в коде (для целей конверсии)

Чтобы цели Метрики срабатывали, нужно добавить в код:

**`src/components/BookingForm.tsx`** (после успешной отправки):
```ts
window.dataLayer?.push({
  event: "form_submit",
  page_type: detectPageType(),
  page_slug: window.location.pathname,
  utm: getUtm(),
});
```

**`src/components/Footer.tsx`** + `Header.tsx` (на кликах WA/TG/tel):
```tsx
<a href="https://wa.me/79185875454" onClick={() => window.dataLayer?.push({ event: "whatsapp_click" })}>
```

**`src/components/RouteFactsLongread.tsx`** — IntersectionObserver на прайс-блок → `scroll_to_price`.

Это работа на ~2 часа. Делается сразу после установки счётчика Метрики.
